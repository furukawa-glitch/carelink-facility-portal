# カイポケ 夜間バッチ取込仕様（プレースホルダ）

`docs/batch_overnight_pipeline.md` のフェーズ5で出力する CSV/API の**列定義・文字コード**を、実環境のカイポケ仕様に合わせてここに記載する。

- 本ファイルは**機微情報を含めない**。
- 確定後、`operations/export_kaipoke_batch.py` を実装する。

## 想定フィールド（たたき台）

| 列名（仮） | 内容 |
|------------|------|
| patient_external_id | カイポケ側IDまたは内部 patient_id |
| visit_date | 訪問日 |
| start_time / end_time | 開始・終了 |
| soap_s / soap_o / soap_a / soap_p | 下書き（確定前フラグ付きでも可） |

---

## 観測済み：看護記録書Ⅱ 一括CSV（エクスポート形式の参考）

オーナー提供の `看護記録書Ⅱ_2026年03月01日-2026年03月31日.csv` に基づく**列名の一覧**（値は記載しない）。  
夜間バッチでカイポケへ**再取込・突合**する場合、**同一列順・同一ヘッダ**を目指すと差分が少ない。

### 先頭列（識別・訪問枠）

- `訪問日`, `利用者名`, `利用者カナ`, `緊急訪問`, `予定外訪問`, `訪問開始時刻`, `訪問終了時刻`, `サービス内容`, `精神科`, `職員名1`〜`3`, `緊急訪問の理由`

### バイタル（最大3回分）

- `1回目：体温（℃）` … `3回目：バイタルサインコメント` まで（様式どおり）

### 観察・ケア（チェック項目多数）

- `意識状態`, `浮腫`, `浮腫コメント`, 睡眠・排便・排尿・疼痛 等、清拭・入浴・排泄・食事・服薬・環境整備、**医療の実施：**〜、**管理・指導：**〜、リハ項目、精神科項目、写真欄、`最終更新日時`, `最終更新者`

### ナラティブ（監査エンジン入力に使える列）

- **`観察記録`** — SOAP 相当の本文が入る。**`operations/audit_check_engine.py`** は、行単位でこの列をテキストファイルに切り出してチェック可能。

### 文字コード・区切り

- 実ファイルは **ダブルクォート付きCSV**, 先頭行ヘッダ。文字コードは環境により **UTF-8** または **CP932** のため、取込時は `--encoding` を明示すること。

### バッチでの使い方（案）

1. **エクスポートCSV** を `data/` 外で保管し、分析・監査のみに使う。  
2. 録音から生成した下書きを**同じ列構造**にマッピングして `export_kaipoke_batch.py` から出力する。  
3. 利用者別にフィルタする場合は `利用者名` 列ではなく、可能なら **内部 patient_id** 列が追加されるようカイポケ側設定を検討（プライバシー・突合作業のため）。
