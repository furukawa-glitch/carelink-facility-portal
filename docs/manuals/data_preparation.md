# データ準備マニュアル — 勤務表 × カイポケ実績の照合用

ケアサポート向け「勤務実績の自動照合」で使うファイルの形式をそろえるための手順です。  
プログラム（`operations/attendance_checker.py`）は、ここで定める**標準列名**を前提に読み込みます。

---

## 1. 全体の考え方

- **突合の単位**：原則 **1 日 × 1 スタッフ × 1 勤務ブロック**（同日に複数行がある場合は、後述のルールに従う）。
- **照合する時刻**：**出勤（始業）**と**退勤（終業）**の2点。休憩は第1版では対象外とする（必要になったら版を上げる）。
- **ズレの扱い**：出勤または退勤のどちらかについて、勤務表とカイポケの差が **1 分以上** あれば「要確認」としてリストに載せる。

---

## 2. ファイル名の規則（推奨）

運用で迷わないよう、次の形式を推奨します。

| 種別 | 命名例 | 備考 |
|------|--------|------|
| 勤務表（Excel） | `勤務表_YYYYMM_ケアサポート.xlsx` | 月次まとめて1ファイルでも、週次で複数でも可 |
| カイポケ実績（CSV） | `カイポケ_勤務実績_YYYYMMDD_HHMM.csv` | エクスポートした日時をファイル名に含めると履歴管理しやすい |

- ファイル名に個人名を入れない（個人情報の混入防止）。
- 同じ月の照合なら、**勤務表とCSVの対象期間が一致しているか**を照合前に人の目で確認する。

### 2.1 プロジェクト内の保存場所（CareLink_AI 標準）

照合用の**最新の2ファイル**は、次のフォルダに置く（パスは `CareLink_AI` フォルダを開いたときの相対パス）。

| 保存先フォルダ | 用途 |
|----------------|------|
| `data/attendance_inbox/` | 勤務表 Excel と カイポケ CSV を置く（照合の入り口） |

**ファイル名の例（2026年4月分を照合する場合）**

| 種類 | 保存するファイル名の例 |
|------|------------------------|
| 勤務表 | `勤務表_202604_ケアサポート.xlsx` |
| カイポケ実績 | `カイポケ_勤務実績_20260411_1530.csv` |

- `202604` は**照合したい年月**（YYYYMM）。`20260411_1530` は**CSVを書き出した日時**（任意だが、上書き混乱防止のため推奨）。
- 週次で複数ある場合は `勤務表_202604_第2週_ケアサポート.xlsx` のように週番号を足してよい（拡張子は `.xlsx` のまま）。

## 3. 勤務表（Excel）の形式

### 3.1 ファイル形式

- **拡張子**：`.xlsx`（Excel ブック）
- **シート名**：`勤務データ`（既定）。別名にする場合は `attendance_checker.py` の `EXCEL_SHEET_NAME` を変更する。

### 3.2 必須列（ヘッダー行 1 行目）

プログラムが読む**標準列名**は次のとおりです（セル内の文字を**一字一句**合わせる）。

| 列名（標準） | 内容 | 例 |
|--------------|------|-----|
| `日付` | 勤務日 | `2026/4/1` または `2026-04-01` |
| `スタッフコード` | 照合の主キー（推奨） | `CS-00123` |
| `スタッフ氏名` | コードが無い場合の補助・確認用 | `山田 太郎` |
| `出勤時刻` | 始業 | `9:00` / `09:00:00` |
| `退勤時刻` | 終業 | `17:30` |

### 3.3 列が元帳と違う場合

- Excel の列名が `開始` `終了` など別名のときは、**照合前に Excel 側で列名を上表の標準名にリネーム**する（コピー用シートを作ってもよい）。
- **スタッフコード**が無い運用の場合は、**氏名と日付のみ**で突合する設定にプログラム側で切り替えられるが、同名の別人がいる拠点では誤判定のリスクがあるため、コードの付与を推奨する。

### 3.4 同日複数行（シフト分割）

- 原則：**カイポケ側の行と 1 対 1 で対応する行**になるよう、Excel も同じ粒度で行を分ける。
- どうしても片方だけが1行にまとまっている場合は、第1版では自動照合から外し、マニュアルで「要手作業」とする。将来、複数行マージ規則を定義してから対応する。

### 3.5 時刻・日付の注意

- **日付と時刻は分離**：日付列は日付のみ、時刻列はその日の出勤・退勤（24時間表記推奨）。
- 夜勤で**日をまたぐ退勤**の場合は、社内ルールを一つに決める（例：退勤日付を「出勤日」と同じ行に書く／翌日の日付行に書く）。ルールが混在すると自動照合が狂うため、**ケアサポート内で統一**する。

## 4. カイポケ CSV の形式

### 4.1 文字コードと区切り

- **文字コード**：`UTF-8`（推奨）。Shift-JIS でエクスポートされる場合は、Excel で UTF-8 に保存し直すか、プログラムの `CSV_ENCODING` を `cp932` に変更する。
- **区切り文字**：カンマ `,`（CSV 既定）。タブ区切りの場合はプログラムの読み込みオプションを変える必要がある。

### 4.2 必須列（標準列名への対応）

カイポケのエクスポート列名は契約・メニューで異なることがあります。**実際に出た CSV の1行目（ヘッダー）を見て**、次の標準列に相当する列を特定し、**CSV 上で列名をリネーム**するか、プログラム内の `KAIPOKE_COLUMN_ALIASES` に「元の列名 → 標準名」の対応を追記する。

プログラムが期待する**標準列名**：

| 列名（標準） | 意味 |
|--------------|------|
| `日付` | 勤務日 |
| `スタッフコード` | 勤務表と同じコード体系が望ましい |
| `スタッフ氏名` | コード突合の補助 |
| `出勤時刻` | 打刻または実績の始業 |
| `退勤時刻` | 打刻または実績の終業 |

カイポケ側で「利用者向けの記録」と「スタッフ勤務」が混在するエクスポートの場合は、**スタッフ勤務だけが残るようフィルタした CSV** を別名で保存してから照合する。

### 4.3 サンプルで確認すること（チェックリスト）

- [ ] 1 行が「1 スタッフ・1 日・1 勤務」に相当しているか。
- [ ] 日付・時刻が文字列として崩れていないか（Excel で開いて列が数値化されていないか）。
- [ ] スタッフコードが勤務表と**同じ桁・同じ表記**か（先頭ゼロの有無など）。

## 5. 個人情報・セキュリティ

- リポジトリや共有ドライブに置くファイルは、**必要最小限の期間**にとどめる。
- Git にコミットする場合は、**実データを含めない**（サンプルは架空データのみ）。`.gitignore` で `*.xlsx` `*_実績*.csv` 等を除外する運用を推奨する。

---

## 6. 実行手順（概要）

1. 勤務表 Excel を本マニュアルどおり整形する。
2. カイポケから CSV を出力し、列を標準に合わせる（またはエイリアスを設定する）。
3. 仮想環境で依存パッケージを入れる：  
   `pip install pandas openpyxl`
4. 照合実行（`CareLink_AI` フォルダで実行する例）：  
   `python operations/attendance_checker.py --shift data/attendance_inbox/勤務表_202604_ケアサポート.xlsx --kaipoke data/attendance_inbox/カイポケ_勤務実績_20260410_1530.csv`

出力された「要確認リスト」を人事・現場で確認し、カイポケまたは勤務表のどちらを正とするかを決める。
