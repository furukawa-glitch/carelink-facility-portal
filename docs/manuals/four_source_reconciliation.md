# 4ソース照合 — 勤務表・出勤簿・カイポケ・ライン表

`operations/reconcile_four_sources.py` が、**4ファイルを読み取るだけ**で突き合わせ、結果を **`data/reports/` に新規 Excel のみ**書き出します。**原本は開いても保存しません（読み取り専用）**。

---

## 入力ファイル（標準列）

すべて **1行＝1スタッフ×1日** の一覧形式を推奨します（訪問が複数行のカイポケは `visit_csv` モードで日次集約）。

### 共通（勤務表・出勤簿・カイポケフラット・ライン表）

| 列名 | 必須 | 内容 |
|------|------|------|
| `日付` | ○ | `2026-03-01` 形式など |
| `スタッフ氏名` | ○ | 突合の主軸（コード併用可） |
| `スタッフコード` | △ | ある場合は `--use-staff-code` でコード優先突合 |
| `出勤時刻` | △ | 時刻比較に使用（空なら時刻系の不一致は出にくい） |
| `退勤時刻` | △ | 同上 |

### ライン表の追加列

| 列名 | 必須 | 内容 |
|------|------|------|
| `出欠`（列名は `--line-status-col` で変更可） | 推奨 | `出勤` 相当の値で「出勤」とみなす（`--line-present-values`） |

「出勤」に相当する値の例（既定）: `出勤`, `○`, `有`, `出`, `予定出勤`

### カイポケ（訪問スケジュールCSV・1行1訪問）

`--kaipoke-mode visit_csv` のとき:

- `職員名１` … スタッフ名
- `日付` … **月内の日**（1〜31 の数値）
- `開始時間` / `終了時間` … 訪問枠（同一職員同一日で **最早開始・最遅終了** に集約して「実績」とする）

`--kaipoke-year-month` に `202603` のように **YYYYMM** を指定する。

---

## 検出する不一致（例）

| 種別 | 条件（概要） |
|------|----------------|
| 勤務表 vs 出勤簿 | 両方に時刻があり、出勤または退勤が **閾値秒以上** ずれる（既定60秒＝1分） |
| 勤務表 vs カイポケ | 同上 |
| 出勤簿 vs カイポケ | 同上 |
| ライン vs カイポケ | ラインが「出勤」なのに、カイポケ側に **その日の実績（時刻）がない** |
| 片方のみ行がある | どれか1ソースにしか行がない（キー: 日付＋スタッフ） |

---

## 実行例

```text
cd CareLink_AI
pip install pandas openpyxl
python operations/reconcile_four_sources.py ^
  --shift data/attendance_inbox/勤務表_202603_ケアサポート.xlsx ^
  --attendance data/attendance_inbox/出勤簿_202603.xlsx ^
  --kaipoke data/attendance_inbox/訪問看護スケジュール_202603.csv ^
  --kaipoke-mode visit_csv ^
  --kaipoke-year-month 202603 ^
  --line data/attendance_inbox/ライン表_202603.xlsx ^
  --line-sheet ライン表
```

カイポケが **一覧形式の xlsx** のときは `--kaipoke-mode flat` のまま、`--kaipoke-sheet` でシート名または番号を指定できる。

出力: `data/reports/reconcile_20260411_153045.xlsx`（実行日時でファイル名が変わる）

---

## 注意

- 氏名の空白・全角半角の違いで未マッチしやすい。可能なら **スタッフコード** を全ソースで統一する。
- 原本の Excel/CSV は **本ツールが上書きしません**。編集しないでください（運用ルール）。
